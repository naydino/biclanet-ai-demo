Hardware & Sensor Functions

Detailed specifications on how each component operates to ensure system reliability and security.

Lock Mechanism

  • Controls the physical locking arm for each bike.
  • Reports status in real-time: Locked, Unlocked, Error.
  • Supports remote actions like Force Lock and Emergency Open.

RFID / QR Access

  • Riders authenticate at the pod using the mobile app or an RFID card.
  • Access tokens are securely verified against the BiclaNet Cloud in real-time.
  • Unlock command is triggered only after a valid reservation check.

Bike Presence Sensor

  • Infrared or ultrasonic sensor verifies a bike is physically present.
  • Prevents false occupancy and ghost reservations.
  • Feeds true usage and dwell time data to the analytics dashboard.

Environment Sensors (Optional)

  • Measures ambient temperature and humidity for operational insights.
  • Can be expanded to include air quality sensors (PM2.5, NO2).
  • Data supports environmental comfort metrics and city reporting dashboards.

Dashboard, App, AI & Safety

Dashboard & App Integration

Both the operator console and the rider app read from the same real-time data source, ensuring data consistency and immediate updates for all users.

  • Web Console: Shows live pod status, occupancy, errors, and history.
  • Rider App: Shows availability, reservations, and unlock access.

AI & ML Layer

Historical usage data feeds machine learning models that power intelligent predictions and optimizations across the entire BiclaNet ecosystem.

  • Dashboard AI optimizations for resource allocation.
  • Occupancy forecasting for peak demand planning.
  • Rider app tips and real-time availability forecasts.

Safety & Fail-Safes

Built-in redundancy and fail-safe mechanisms ensure continuous operation even during network outages or hardware malfunctions.

  • Local manual override during network failure.
  • Error state detection (low voltage, failed lock).
  • Automatic alerting inside the dashboard system log.

Installation & Maintenance

  • Site Selection: Pods should be installed in areas with stable WiFi connectivity or cellular coverage for reliable cloud communication.
  • Power Requirements: Standard AC power (110-240V) required. Optional solar panels available for remote installations.
  • Mounting: Wall-mounted or freestanding units available. Follow manufacturer installation guidelines for structural requirements.
  • Initial Setup: Pods are pre-configured with unique IDs. Register pods in the operator dashboard during initial deployment.
  • Regular Maintenance: Monthly visual inspections recommended. Lock mechanisms should be lubricated quarterly. Sensor calibration checks every 6 months.
  • Firmware Updates: Over-the-air updates managed automatically through the operator dashboard. Pods remain operational during updates.

Data & AI Insights

The BiclaNet platform collects real-time occupancy data, usage patterns, and environmental metrics from each pod. This data powers AI-driven predictions that help operators optimize fleet management:

  • Usage Predictions: Machine learning models analyze historical data to forecast demand patterns, helping operators anticipate peak usage times and plan maintenance windows.
  • Occupancy Forecasts: 24-hour occupancy predictions enable proactive pod placement and bike redistribution strategies.
  • Maintenance Alerts: AI models detect anomalies in sensor readings, lock operations, and connectivity patterns to flag potential hardware issues before they impact users.
  • Optimization Recommendations: The system suggests pod placement adjustments, capacity optimization, and usage-based routing improvements based on aggregated fleet data.

Support & Troubleshooting

Pod shows as offline in the dashboard

Check WiFi connectivity or cellular signal strength at the pod location. Verify power supply. If connectivity is stable, try rebooting the pod via the dashboard's remote reboot feature. Contact support if the issue persists after 24 hours.

Lock mechanism not responding

First, attempt a remote unlock via the dashboard. If unsuccessful, check for physical obstructions. Lock mechanisms may require manual override in emergency situations. Schedule on-site maintenance if the lock continues to malfunction.

Incorrect occupancy readings

Sensor calibration may drift over time. Use the calibration tool in the pod control page to reset sensor baselines. Ensure the pod interior is clean and free of debris that might interfere with presence detection. If readings remain inaccurate, sensor replacement may be required.

RFID/QR reader not scanning

Verify that the reader module is powered and the LED indicator is active. Clean the reader surface. Check that RFID cards are properly formatted and QR codes are not damaged. If multiple users report issues, the reader module may need replacement.

Firmware update failed

Pods will automatically retry failed updates. If updates consistently fail, check network connectivity and available storage. Pods maintain the previous firmware version if an update fails, so operation continues normally. Contact support for manual update assistance if automatic retries are unsuccessful.

How do I access pod logs?

Navigate to the Pod Control page for any individual pod and use the "Export Logs" button to download a detailed log file. Logs include sensor readings, lock operations, access attempts, and system events for the selected time period.
